Welcome to DBA, the FI-WARE DB Anonymizer Generic Enabler reference implementation.

This project hosts the source code of DBA. This code has been developed thanks to the support of the FI-WARE project, an initiative funded by the European Commission. Please refer to the FI-WARE website for more information.

DBA is a service that helps in data disclosure activities, and in particular when considering, evaluating and applying anonymization strategies. DB Anonymizer DBA GE Reference implementation is designed to be easily reused by other services, compositions or applications.

DBA is an implementation of the DB Anonymizer GE specification.

Why do I need DBA?

DBA permits to understand if a certain anonymization policy for a dataset should be considered safe or not. The service offers a function that calculates a value, that represents the possibility of data that an attacker can reconstruct from an anonymized dataset content (that is, the re-identification risk).

Documentation

Quick Reference Guide

DBA has a very simple API. Its main functionality can be used through two simple methods, one to submit a policy to be analysed (together with its associated dataset), the other to retrieve the computation result. Other functionalities exist, please refer to this link for a list of functionalities and a more detailed explanation.

In this example, only the main functionality is demonstrated, the others can be invoked in a similar way.

  1. submitting a policy:
curl \ 
> -F "policyFile=@.xml" \
> -F "dbDump=@.zip" \
> -F "submit=Submit" https://[server URL]/[application path]/DBA/evaluatePolicy
  1. retrieving re-identification risk for that policy:
curl https://[server URL]/[application path]/DBA/getPolicyResult/[GID returned by the previous call]
